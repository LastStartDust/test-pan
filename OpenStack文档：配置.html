local.conf中¶

通过文件修改DevStack配置local.conf。它是一个修改后的INI格式文件，它引入了一个元部分标题来传送关于要更改的配置文件的附加信息。

提供样品 devstack/samples

新标题类似于普通的INI部分标题，但是使用双括号（）和两个内部字段（由pipe（）分隔）。请注意，双括号和内部字段之间没有空格。同样，管道和内部区域之间也没有空格：[[ ... ]]|

'[['  < phase >  '|'  < config - file - name >  ']]'

其中<phase>是由一组相位名称定义的stack.sh ，<config-file-name>是配置文件名。文件名在stack.sh上下文中进行评估，因此所有环境变量都可用，可以使用。强烈建议在标题中使用项目配置文件变量（请参见NOVA_CONF下面的示例）。如果配置文件的路径不存在，将被跳过。

定义的阶段是：

  • 地方 -提取localrc从local.conf之前 stackrc的来源
  • post-config - 在第2层服务配置之后并在启动之前运行
  • 服务启动后，extra.d执行任何文件之前，会额外运行
  • post-extra - 在extra.d执行文件之后运行

文件严格按顺序处理; 元部分可以被指定一次以上，但如果任何设置被重复，最后出现在文件中将被使用。

[-配置后| $ NOVA_CONF]
[默认]
use_syslog = True

[osapi_v3]
enabled = False

一个特定的元节local|localrc用于提供一个默认 localrc文件（实际上.localrc.auto）。这允许DevStack的所有自定义设置都包含在单个文件中。如果localrc 存在，将使用它来保留向后兼容性。

[本地| localrc]
IPV4_ADDRS_SAFE_TO_USE = 10.254.1.0 / 24
ADMIN_PASSWORD = SPECIALE
LOGFILE = $ DEST /日志/ stack.sh.log

注意，这Q_PLUGIN_CONF_FILE是唯一的，因为它被假定 不以/（斜线）字符开头。需要添加斜杠：

[-配置后| / $ Q_PLUGIN_CONF_FILE]

还要注意，该localrc部分是作为一个shell脚本片段，必须符合shell要求，特别是=（等于）周围没有空格。

OpenRC的¶

openrc配置适用于OpenStack命令行工具的登录凭据。openrc源stackrc（起源于其localrc部分 local.conf），以便 在端点中接收HOST_IP和/或SERVICE_HOST使用。下面显示的值是默认值。

OS_PROJECT_NAME（OS_TENANT_NAME）

Keystone将项目一词标准化为拥有资源的实体。在某些地方，上一个租户对这一用途仍然存在引用 。此外，project_name优先于 project_id。仍然支持OS_TENANT_NAME与旧版工具的兼容性。

OS_PROJECT_NAME = 演示
OS_USERNAME

除了拥有的实体（项目）外，OpenStack还会调用执行操作用户的实体。

OS_USERNAME = 演示
OS_PASSWORD

Keystone的默认身份验证需要提供密码。关于将密码置于环境变量中的通常注意事项适用，对于大多数DevStack使用，这可能是可接受的折衷。

OS_PASSWORD = 秘密
HOST_IP，SERVICE_HOST

设置API端点主机使用HOST_IP。SERVICE_HOST也可以用于指定端点，这对于某些local.conf配置是方便的 。通常，HOST_IP在该localrc部分中设置 。

HOST_IP = 127.0.0.1
SERVICE_HOST = $ HOST_IP
OS_AUTH_URL

使用Keystone对OpenStack云进行身份验证返回 令牌和服务目录。目录包含用户/租户可以访问的所有服务的端点 - 包括Nova，Glance，Keystone和Swift。

OS_AUTH_URL = HTTP：// $ SERVICE_HOST：5000 / V2.0
KEYSTONECLIENT_DEBUG，NOVACLIENT_DEBUG

将命令行客户端日志级别设置为DEBUG。默认情况下，这些被注释掉。


＃export KEYSTONECLIENT_DEBUG = 1 ＃export NOVACLIENT_DEBUG = 1

最小配置¶

虽然stack.sh很高兴没有一个localrc部分 local.conf，devlife更好，当有一些最小的变量设置。这是一个触发最常需要设置的值的最小配置的示例。

  • 没有记录
  • 预先设置密码以防止交互提示
  • 将网络范围远离本地网络（IPV4_ADDRS_SAFE_TO_USE 并FLOATING_RANGE在下面注释）
  • 如果检测不可靠HOST_IP，请设置主机IP（以下注释掉）
[本地| localrc]
ADMIN_PASSWORD =秘密
DATABASE_PASSWORD = $ ADMIN_PASSWORD
RABBIT_PASSWORD = $ ADMIN_PASSWORD
SERVICE_PASSWORD = $ ADMIN_PASSWORD
＃IPV4_ADDRS_SAFE_TO_USE = 172.31.1.0 / 24
＃FLOATING_RANGE = 192.168.20.0 / 25
＃HOST_IP = 10.3.4.5

如果在*_PASSWORD此处未设置变量，系统将提示您为其输入值stack.sh。

网络范围不得与主机上正在使用的任何网络重叠。重叠并不罕见，因为RFC-1918的“私有”范围通常用于本地网络和Nova的固定和浮动范围。

HOST_IP通常在第一次运行时检测到，stack.sh但是由于IP从以太网接口移动到主机上的桥，因此在后续运行中通常是不确定的。在这里设置它也使它可以openrc设置OS_AUTH_URL。HOST_IP默认情况下不设置。

HOST_IPV6通常在第一次运行时检测到，stack.sh但是如果在默认以太网接口上没有IPv6地址，则不会被设置。在这里设置它也使它可以openrc设置OS_AUTH_URL。 HOST_IPV6默认情况下不设置。

对于与x86默认值不同的架构特定配置，请参阅arch-configuration。

服务回收¶

用于检查每个服务的源的Git存储库由为每个服务设置的一对变量控制。 *_REPO 指向存储库并*_BRANCH选择要检出哪个分支。这些可能会被覆盖，local.conf以从不同的回购中抽取来源进行测试，例如Gerrit分支机构提案。GIT_BASE指向主存储库服务器。

NOVA_REPO = $ GIT_BASE / OpenStack的/ nova.git
NOVA_BRANCH =主

从Gerrit直接拉一个分支，从Gerrit评论页面获取回购和分支：

git  fetch  https ：// review 。OpenStack的。组织/ p / 开栈/ 新星 参/ 更改/ 50 / 5050 / 1  &&  git的 结帐 FETCH_HEAD

回购是以下节fetch，分支是以下节：

NOVA_REPO = https ：// review 。OpenStack的。组织/ p / 开栈/ 新星
NOVA_BRANCH = 参/ 更改/ 50 / 5050 / 1

安装目录¶

DevStack安装目录由DEST变量设置。默认是/opt/stack。

通过在该localrc部分的早期设置，您可以在后面的变量中引用它。设置它即使没有从默认值更改也是有用的。

DEST = / opt / stack

记录¶

启用日志记录¶

默认情况下，stack.sh输出仅写入运行的控制台。通过设置LOGFILE到目标日志文件的全限定名称，可以将其发送到控制台以外的 文件。每个运行的时间戳将附加到给定的文件名 stack.sh。

LOGFILE = $ DEST /日志/ stack.sh.log

如果LOGDAYS设置为要保留的旧日志文件的天数，旧的日志文件将自动清除。

LOGDAYS = 1

在DevStack运行期间使用一些着色，以便更容易地看到发生了什么。这可以通过以下方法禁用：

LOG_COLOR = False

记录服务输出¶

默认情况下，服务在systemd本地登录到系统日志。

要查询日志使用journalctl命令，例如：

journalctl  - 单元 devstack @ *

查询日志中可以找到更多的例子。

日志记录配置示例¶

例如，非交互式安装可能希望将输出保存到文件，保留服务日志并禁用存储文件中的颜色。

[本地| localrc]
DEST = /选择/堆叠/
LOGFILE = $ LOGDIR / stack.sh.log
LOG_COLOR =假

数据库后端¶

有多个数据库后端可用。可用的数据库在lib / databases目录中定义。 mysql是默认数据库，通过在以下localrc部分中放入以下内容来选择其他数据库：

disable_service  mysql 
enable_service  postgresql

mysql 是默认数据库。

RPC后端¶

包括RabbitMQ RPC后端的支持。其他RPC后端可能通过外部插件提供。启用或禁用RabbitMQ是通过常规的服务功能来处理的 ENABLED_SERVICES。

禁用RabbitMQ的示例local.conf：

disable_service  兔子

Apache前端¶

可以为支持在HTTPD + mod_wsgi下部署的wsgi服务启用Apache Web服务器。默认情况下，建议在HTTPD + mod_wsgi下运行的服务部署在Apache下。要使用该支持HTTPD +替代mod_wsgi的一组服务的替代部署策略（如eventlet） ENABLE_HTTPD_MOD_WSGI_SERVICES到False你 local.conf。

可以在HTTPD + mod_wsgi下运行的每个服务都有一个可以在你的设置中设置的覆盖切换local.conf。

mod_wsgi默认情况下，Keystone在Apache 下运行。

示例（Keystone）

KEYSTONE_USE_MOD_WSGI = “True”

示例（Nova）：

NOVA_USE_MOD_WSGI = “True”

示例（Swift）：

SWIFT_USE_MOD_WSGI = “True”

示例（热）：

HEAT_USE_MOD_WSGI = “True”

示例（煤渣）：

CINDER_USE_MOD_WSGI = “True”

从Git的库¶

默认情况下，devstack从git安装OpenStack服务器组件，但是它会在pypi上从已发布的版本安装客户端库。如果您正在从事服务器开发工作，那么这是适当的，但是如果您想了解客户端未发行版本如何影响系统，则可以通过在上游或通过指定本地git树将其从上游安装到devstack LIBS_FROM_GIT。多个库可以指定为逗号分隔列表。

LIBS_FROM_GIT = python - keystoneclient ，oslo 。配置

将变量设置为ALL激活所有库的下载。

虚拟环境¶

通过设置启用使用Python虚拟环境USE_VENV 来True。这将为PROJECT_VENV数组中定义的每个项目创建静态图像。

PROJECT_VENV数组中的每个条目都包含用于项目的venv的目录名称。数组索引是项目名称。如果需要，多个项目可以使用相同的文件。

PROJECT_VENV [ “一瞥”] = $ {GLANCE_DIR} .venv

ADDITIONAL_VENV_PACKAGES是将要安装到每个venv的附加包的逗号分隔列表。通常，项目不会在其requirements.txt文件中列出某些包，因为它们是“可选”要求，即仅在某些配置中需要。默认情况下，启用的数据库将在启用时添加其Python绑定。

ADDITIONAL_VENV_PACKAGES = “python-foo，python-bar”

每次干净安装¶

默认情况下，stack.sh仅克隆项目回馈（如果不存在）$DEST。stack.sh如果RECLONE设置为每次运行，将会更新每个回购 yes。这样可以避免手动删除回收以获取当前分支$GIT_BASE。

RECLONE = 是

升级软件包由¶ 安装

默认情况下stack.sh只安装Python包，如果当前没有安装版本，或者当前版本与指定的要求不匹配。如果PIP_UPGRADE设置为，True那么现有的所需Python包将被升级到与要求相匹配的最新版本。

PIP_UPGRADE = True

嘉宾图片¶

通过逗号分隔的IMAGE_URLS 变量在URLS中提供的图像将被DevStack下载并上传。

默认的guest虚拟机映像是针对每种类型的虚拟机管理程序及其测试要求进行预定义的stack.sh。设置 DOWNLOAD_DEFAULT_IMAGES=False将阻止DevStack下载这些默认图像; 在这种情况下，您将需要填充 IMAGE_URLS足够的图像以满足测试要求。

DOWNLOAD_DEFAULT_IMAGES = False 
IMAGE_URLS = “http://foo.bar.com/image.qcow”，
IMAGE_URLS + = “http://foo.bar.com/image2.qcow”

实例类型¶

DEFAULT_INSTANCE_TYPE可用于配置默认实例类型。当没有指定此参数时，Devstack为真正的小型实例创建了额外的微量和纳米风味来运行Tempest测试。

对于具有较大内存要求的客户端，DEFAULT_INSTANCE_TYPE 应在配置文件中指定，因此Tempest会选择默认密码。

KVM on Power与QEMU 2.4需要512 MB来加载固件 - QEMU 2.4 - PowerPC，因此在ppc64 / ppc64le上运行实例的用户可以选择默认创建的风格之一，如下所示：

DEFAULT_INSTANCE_TYPE = m1 。小

IP版本¶

IP_VERSION可用于配置中子通过创建是IPv4，IPv6或双栈自助服务项目数据网络与任一IP_VERSION=4，IP_VERSION=6或IP_VERSION=4+6 分别。

IP_VERSION = 4 + 6

以下可选变量可用于更改默认IPv6行为：

IPV6_RA_MODE = SLAAC
IPV6_ADDRESS_MODE = SLAAC
IPV6_ADDRS_SAFE_TO_USE = FD $ IPV6_GLOBAL_ID :: / 56
IPV6_PRIVATE_NETWORK_GATEWAY = FD $ IPV6_GLOBAL_ID :: 1

注：IPV6_ADDRS_SAFE_TO_USE并IPV6_PRIVATE_NETWORK_GATEWAY 可以与任何有效的IPv6前缀进行配置。默认值使用自动生成IPV6_GLOBAL_ID符合RFC4193。

服务版本¶

DevStack可通过设置使得能够在IPv4或IPv6服务操作SERVICE_IP_VERSION要么SERVICE_IP_VERSION=4或 SERVICE_IP_VERSION=6分别。

当设置为4devstack服务将打开监听套接字 0.0.0.0，服务端点将使用HOST_IP 地址注册。

当设置为6devstack服务将打开监听套接字:: ，服务端点将使用HOST_IPV6地址注册。

此设置的默认值为4。例如，4+6目前不支持双模式支持。 HOST_IPV6可以选择用于更改默认的IPv6地址

HOST_IPV6 = $ {} some_local_ipv6_address

斯威夫特¶

默认情况下，Swift已禁用。启用时，它只配置一个副本，以避免在小型虚拟机上IO /内存密集。

如果您想启用Swift，您可以将其添加到您的localrc 部分：

enable_service  s - proxy  s - object  s - container  s - account

如果您只想使用Swift和Keystone安装Swift，您可以在您的localrc部分中进行安装：

disable_all_services 
enable_service  key  mysql  s - proxy  s - object  s - container  s - account

如果你只想做一个真正正常的迅速集群多个副本，你可以通过自定义变量这样做的一些测试 SWIFT_REPLICAS在你的localrc部分（通常是3）。

您可以手动覆盖环形建筑以使用特定的存储节点，例如，当您要测试多节点环境时。在这种情况下，您必须SWIFT_STORAGE_IPS在您的localrc部分中设置应用作Swift存储节点的空格分隔的IP列表 。请注意，这不会创建多节点设置，它仅在将节点添加到Swift环时使用。

SWIFT_STORAGE_IPS = “192.168.1.10 192.168.1.11 192.168.1.12”
Swift S3¶

如果您在DevStack 中启用swift3，ENABLED_SERVICES则将安装swift3中间件仿真。Swift将被配置为充当Keystone的S3端点，从而有效地替换 nova-objectstore。

只有Swift代理服务器在systemd系统中启动，所有其他服务都将在后台启动并由swift-init工具进行管理。

暴风¶

如果tempest已成功配置，则可以按以下方式运行基本的烟雾测试：

$ cd / opt / stack / tempest
$ tox -efull tempest.scenario.test_network_basic_ops

默认情况下，下载tempest并生成配置文件，但tempest软件包未安装在系统的全局站点软件包中（软件包安装包括安装依赖项）。所以暴风雨不会在毒素之外运行。如果要安装，请在您的localrc部分添加以下 内容：

INSTALL_TEMPEST = True

细胞¶

单元格是一种替代缩放选项。要设置单元格环境，请在您的localrc部分中添加以下内容：

enable_service  n - 单元格

请注意，目前在单元格中缺少一些功能，一个值得注意的是安全组。修补练习已禁用单元格不支持的功能。

煤渣¶

用于保存Cinder管理卷VOLUME_GROUP_NAME的逻辑卷组由以下设置，逻辑卷名称前缀设置为， VOLUME_NAME_PREFIX并且卷背景文件的大小设置为VOLUME_BACKING_FILE_SIZE。

VOLUME_GROUP_NAME = “堆栈卷” 
VOLUME_NAME_PREFIX = “volume-” 
VOLUME_BACKING_FILE_SIZE = 10250 M

梯形¶

多区域设置¶

我们要用共享梯形失真（相同的用户和服务）和地平线设置两个devstack（RegionOne和RegionTwo）。Keystone和Horizo​​n将位于RegionOne。完整规格可在以下网站获取：https： //wiki.openstack.org/wiki/Heat/Blueprints/Multi_Region_Support_for_Heat。

在区域一：

REGION_NAME = RegionOne

在区域二：

disable_service  horizo​​n 
KEYSTONE_SERVICE_HOST = < KEYSTONE_IP_ADDRESS_FROM_REGION_ONE > 
KEYSTONE_AUTH_HOST = < KEYSTONE_IP_ADDRESS_FROM_REGION_ONE > 
REGION_NAME = RegionTwo 
KEYSTONE_REGION_NAME = RegionOne

在RegionOne的devstack中，我们将REGION_NAME设置为RegionOne，因此在此Devstack中启动的服务区域注册为RegionOne。在RegionTwo的devstack中，同样地，我们将REGION_NAME设置为RegionTwo，因为我们希望在此Devstack中启动的服务在RegionTwo中注册。但是，Keystone服务已启动并注册到RegionOne，而不是RegionTwo，因此我们使用KEYSTONE_REGION_NAME来指定Keystone服务的区域。KEYSTONE_REGION_NAME的默认值与REGION_NAME相同，因此我们在RegionOne的配置中省略它。

禁用Identity API v2¶

Identity API v2在三鹰市不推荐使用，建议仅使用v3 API。可以在没有v2 API的情况下设置梯形失真：

ENABLE_IDENTITY_V2 = False

练习¶

exerciserc用于配置运动脚本的设置。下面显示的值是默认值。这些都可以通过在localrc部分中设置来覆盖。

  • 最大时间等待，而vm从构建到活动状态

    ACTIVE_TIMEOUT == 30

  • 最长时间等待正确的IP关联和关联。

    ASSOCIATE_TIMEOUT = 15

  • 最大时间直到vm是可引导的

    BOOT_TIMEOUT = 30

  • 运行实例命令直到运行的最大时间

    RUNNING_TIMEOUT = $（（$ BOOT_TIMEOUT + $ ACTIVE_TIMEOUT））

  • 等待vm终止的最长时间

    TERMINATE_TIMEOUT = 30

架构¶

上游CI仅在具有x86架构的节点上运行，但OpenStack支持更多的体系结构。其中一些需要以某种方式配置Devstack。

s390x上的KVM（IBM z系统）¶

Kilo发布后支持s390x（IBM z系统）上的KVM 。对于一体式设置，local.conf文件中的这些最小设置是必需的：

[本地| localrc]
ADMIN_PASSWORD =秘密
DATABASE_PASSWORD = $ ADMIN_PASSWORD
RABBIT_PASSWORD = $ ADMIN_PASSWORD
SERVICE_PASSWORD = $ ADMIN_PASSWORD

DOWNLOAD_DEFAULT_IMAGES =假
IMAGE_URLS = “https://cloud-images.ubuntu.com/xenial/current/xenial-server-cloudimg-s390x-disk1.img”

enable_service n-sproxy
disable_service n-novnc
disable_service etcd3＃https://bugs.launchpad.net/devstack/+bug/1693192

[-配置后| $ NOVA_CONF]

[serial_console]
base_url = ws：// $ HOST_IP：6083 /＃可选

推理：

  • Devstack的默认图像仅为x86，因此我们会停用下载DOWNLOAD_DEFAULT_IMAGES。以上代码中引用的访客图片IMAGE_URLS为例。可能的s390x客人图像的列表不限于此。
  • 该平台不支持像VNC或SPICE这样的图形化控制台。技术原因是平台上缺少帧缓冲区。这意味着我们依赖需要代理服务的替代功能串行控制台n-sproxy。n-novnc为此，我们也禁用VNC的代理。post-config仅当您想在多合一设置之外使用串行控制台时，才需要该部分中的配置。
  • etcd3只要错误报告https://bugs.launchpad.net/devstack/+bug/1693192未解决，则该服务需要禁用 。

 
注意

要对此Devstack一体式运行Tempest，您需要使用未压缩的小于1GB的客户映像。上图的例子大于那个！
